Технология, основанная на точном представлении знаний в виде формальных логических формул с помощью утверждений.
Принцип:
- DOMAINS: Заданный набор сущностей и их признаков, который задаётся через примитивные типы данных, встроенные в компилятор (числа, текст);
- PREDICATES: Пополняемый, но ограниченный набор отношений, которые могут возникать между сущностями и их признаками. Отношения задаются в произвольной текстовой форме, которая является обозначением этого отношения.
Предикаты могут быть: 
 - строгими, т.е. может быть только один экземпляр такого отношения при определённых значениях сущностей.
 - нестрогими (nondeterm), т.е. может быть несколько экзмпляров такого отношения при разных значениях сущностей.
- CLAUSES: Определённые утверждения о фактах, в которых участвуют конкретные сущности, т.е. перечень отношений между сущностями (из набора PREDICATES), при конкретных значениях сущностей (город - "Москва", страна - "Россия").
Т.е. в какие отношения вступают сущности при определённых их значениях.
Утверждения могут быть:
 - фактами - отношения, которые задаются через значения сущностей;
 - правилами - отношения, которые задаются через другие отношения (факты) и формальные логические операции между ними, которые образуют формальные логические цепочки, формирующие логическое утверждение о данном отношении.
 Т.е. это факты, заданные через другие факты и/или наборы фактов.
 Наборы фактов задаются с помощью переменных в соответствии с синтаксисом языка (заглавные буквы).
- GOAL: Запросы к базе знаний - правильны ли утверждения об отношениях между сущностями при их определённых значениях; или получение через переменные значений тех сущностей, при которых утверждения об их отношениях являются правильными (истинными).

Очень нужна там, где требуется точный и предсказуемый вывод на основе имеющихся данных.
Возможные сферы применения: автоматизация производства (например, маршрутизация по линиям конвейера в зависимости целевых показателей - интенсивность, остановки и т.д., юридические вопросы).
В целом практическое применение на основе прецедентов, представленных в виде знания. Если прецеденты соответствуют точно сформулированной области знания. Прецеденты включаются в знание в виде фактов.

Имеет ограниченный уровень сложности, т.е. при достижении определённого уровня сложности, описание будет невозможным для инженера знаний и разрешения противоречий между прецедентами. Это не позволяет расширять знание на пересекающихся или дополнительных областях.
